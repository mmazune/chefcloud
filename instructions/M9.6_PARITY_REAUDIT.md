## Feature A: Operating Hours

| System | Implementation | Gap vs ChefCloud |
|--------|---------------|------------------|
| TastyIgniter | `LocationSchedule` model, `getOpenTime()` / `getCloseTime()` methods | ChefCloud: Clean-room `BranchOperatingHours` with explicit day-of-week |
| easyappointments | `ea_working_plan` JSON column on providers | ChefCloud: Separate model for clarity |
| cal.com | `Availability` with dateRanges + schedule slots | ChefCloud: Simpler per-day open/close |

**Verdict:** MEETS - Standard operating hours pattern implemented

## Feature B: Blackout Windows

| System | Implementation | Gap vs ChefCloud |
|--------|---------------|------------------|

| easyappointments | `ea_breaks` table for provider breaks | ChefCloud: Branch-level blackouts |
| cal.com | `EventType.hidden` or date blocking | ChefCloud: Explicit datetime ranges |

**Verdict:** MEETS - Blackout pattern with datetime ranges
